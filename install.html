Installation

You can easily download the tool in multiple packaging formats using the folder images above, or click the links below for a more detailed explanation

Current stable version is 1.4.0.

Mac

Make sure you run brew update first and then just run brew install q.

Thanks @stuartcarnie for the initial homebrew formula


RPM-Based Linux distributions

Download the version 1.4.0 RPM here here.

Install using rpm -ivh <rpm-name>.

RPM Releases also contain a man page. Just enter man q.


Debian-based Linux distributions

Download the version 1.4.0 Debian package here here.

Install using sudo dpkg -i <package-file-name> or by double clicking the downloaded file in the file manager.

Debian Releases also contain a man page. Just enter man q.


Arch Linux

A PKGBUILD is available in AUR for Arch Linux users. See installing packages in the wiki or use your favorite AUR helper.

This release also contain the q man page. Just enter man q.


Windows (Beta)

You can download the new beta version for an standard Windows installation from here. Please send me any feedback about the installation and open new issues here in github.

Running q in Windows is a bit different, since it's in a compiled format. You need to run q.exe instead of q when running the tool.

Please note that the installation should automatically add q.exe to the system path, so you're supposed to be able to run q.exe from anywhere in the command line. If for some reason you can't run q.exe from any folder, please drop me a line so I can improve the installation.

NOTE: If there is a problem with a missing DLL, please download and install this Microsot Redistributable. I'm planning on adding it to the final Windows installation.

NOTE2: I've tried to make q for Windows standalone, without any need for dependencies except for maybe the Microsoft Redistributable above. This includes python as well - There is no need to have python installed in order to run q.exe.

NOTE3: The addition of q.exe to the path is immediate, but you need to open a new cmd.exe window in order for it to be applied. This means that command line windows which have been opened before the installation will not be able to find q.exe.


Manual installation (very simple, since there are no dependencies)

  1. Download the main q executable from here into a folder in the PATH.
  2. Make the file executable.

    3. Default settings file

    q supports an option file in ~/.qrc or in the working directory (with the name .qrc) which provides defaults for some or all of the command line options. A sample .qrc file with commented-out options is included. Just put it in your home folder and modify it according to your needs.
