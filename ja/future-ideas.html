将来のアイデア

  • python モジュールを通して、q の機能を使用できるようにする。 アルファ版 は既に準備できており、レビューを歓迎しています。
  • 外部 DB との連携
  • SELECT だけでなく、 INSERT や CREATE TABLE SELECT のような機構を提供する。
  • SQL のパースをより良く、スマートなクエリ実行を許可する。

直前にロードされたデータの再利用は一部のみ実装されています。内部 API ではフルサポートしていますが、コマンドラインでの使用は限定的になっています。 もし、同じコマンドを複数クエリとして実行すると (例 q "select AAA" "select BBB" "select CCC" もし、同じファイル内であれば、クエリは直前に実行されたデータに対しての実行となります。 複数コマンドラインのクエリ実行でのデータ再利用は将来的にサポートするつもりです。
