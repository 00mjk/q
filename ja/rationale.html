要旨

スクリーンに写ったテキストファイルを目の前にして、これがデータベースであればいいのになと思ったことはありませんか？私は何度もそのように思ったことがあります。そして、私はついに自分が欲しかったものはデータベースではなく、SQL 言語であることを理解しました。

SQL はデータのための宣言型言語であり、正確にそれがどのようにされるか気にかけることなく何が欲しいかについて定めるのを許します。 これはSQLがとても強力な理由です。なぜなら、それはデータを、ビットとバイトではなく、データ(と文字)とみなすからである。

このツールのゴールは、テキストファイルの世界と SQL の橋渡しを提供することです。

何故他の Linux のツールでは十分でないのか？

標準 Linux ツールは素晴らしく、私はいつも使っています。Linux の全体構想は、それぞれのジョブをこなすために、最高のツールを着回すというものです。 このツールは他のツールで得られる恩恵を損なうことなく、 Linux ツールセットに SQL の力を加えます。事実、私は awk/sed や grep と同じように q と一緒に他の Linux ツールを使っています。

さらにひとつ注意するべきことは、多くの Linux ツールはテキストをデータとしてではなく、テキストとして扱うということです。そういう意味で、SQL が提供するすべてのデータ関連のツール(表現、グルーピング、順序付け、集約など)へのアクセスを提供するメタツールとして q を見ることができます。

思想

このツールは一般的な Linux/Unix のデザイン原則を念頭に置いてデザインされました。もしあなたがこれらのデザイン原則に興味があるならば、驚異的な本を読んでください。http://catb.org/~esr/writings/taoup/ と特に http://catb.org/~esr/writings/taoup/html/ch01s06.html もしあなたが、このツールが動作方法が、原則のうちの何かしらに強く逆らうと信じるならば、私はそれについてのあなたの見方を聞きたいです。
